# Flutter 开发规范与技术选型

本项目为基于 Flutter 3.22+ 与 Dart 3.0+ 构建的交友聊天 App，开发过程中请严格遵循以下技术栈选型与规范。

## 1. 核心架构与框架
- **核心框架**: `Flutter 3.22+`、`Dart 3.0+`
  - *说明*: 保持最新稳定版，适配国内双端机型，保障交互流畅度。
- **状态管理**: `GetX`
  - *说明*: 采用国内主流的一站式解决方案，兼顾状态管理、路由、依赖注入。
- **路由管理**: `GetX 路由`
  - *说明*: 无需额外依赖，与 GetX 状态管理无缝衔接，配置简单。

## 2. UI 与交互
- **UI 组件**: `Material 3` + 自定义主题
  - *说明*: 国内主流做法，兼顾原生质感与个性化，适配国内用户审美。
- **图标**: `iconfont（阿里）`
  - *说明*: 国内主流图标库，资源丰富、免费可用，适配各类场景。

## 3. 核心功能选型
- **即时通讯**: `腾讯IM SDK`
  - *说明*: 国内成熟云服务，无需自建IM服务，稳定高效，适配社交场景。
- **实时画板**: 自研 `CustomPaint`
  - *说明*: 仅需200行代码，完全可控，适配你画我猜实时同步场景。
- **录音/播放**: `record` + `audioplayers`
  - *说明*: 最新活跃库，适配国内机型，录音清晰、播放稳定。
- **图片处理**: `wechat_assets_picker` + `image_cropper`
  - *说明*: 微信风格交互，用户体验友好，支持图片选择、裁剪等核心需求。

## 4. 数据与网络
- **本地存储**: `Hive` + `flutter_secure_storage`
  - *说明*: 分工明确，Hive 存聊天记录，flutter_secure_storage 存 Token，安全高效。
- **网络请求**: `Dio` + `retrofit`
  - *说明*: 国内开发标配，请求封装便捷，支持鉴权、拦截，适配国内接口规范。

## 5. 扩展与运维
- **消息推送**: `极光推送（个人）`
  - *说明*: 按个人场景选择，触达率高，适配国内各大手机厂商。
- **内测分发**: `蒲公英`
  - *说明*: 国内团队常用内测工具，操作便捷，支持快速分发测试。

## 开发者须知
- 在实现相关功能前，请优先采用上述已确定的技术栈。
- 不要引入与上述选型功能重叠的其他第三方库（例如：不要引入 Provider 或 Bloc 进行状态管理，因为已经选定 GetX；不要引入 shared_preferences，因为已经选定了 Hive 和 flutter_secure_storage 等）。
