# Git Commit 提交信息规范
遵循 Conventional Commits 标准，自动分析代码变更并生成规范的提交信息。

## 1. 提交格式（执行规则）
1. 接收提交类型、描述、修改范围、详细说明，自动生成 **Angular 规范 Git 提交信息**。
2. 严格校验格式，自动修复格式/类型/长度问题。
3. 固定格式：
   - 首行：`<类型>(<范围>): <简短描述>（≤50字符）`
   - 空行
   - 详细说明（可选，每行≤72字符，描述具体做了什么修改，为什么修改）
   - 空行
   - 关联 issues（可选，如 `Closes #42`）
4. **项目专属范围 (Scope)**: 
   - 建议在范围中指明是前端 (`flutter`, `ui`, `im`) 还是后端 (`nestjs`, `db`, `socket`)，或者具体模块名。

## 2. 提交类型（强制）
| 类型     | 说明                     | 适用场景                     |
|----------|--------------------------|------------------------------|
| feat     | 新增功能                 | 新增页面、组件、接口、路由等 |
| fix      | 修复 bug                 | 修复渲染、接口、逻辑、样式问题 |
| docs     | 文档修改                 | 注释、规则、README 等更新 |
| style    | 代码格式（不影响功能）| 格式化、缩进、命名修正       |
| refactor | 重构（无功能无 bug 修复）| 组件拆分、逻辑优化、结构调整 |
| perf     | 性能优化                 | 图片压缩、懒加载、接口缓存优化 |
| test     | 测试用例                 | 单元测试、接口测试           |
| chore    | 构建/依赖/配置            | Flutter 依赖升级、NestJS 配置修改 |
| revert   | 回滚提交                 | 撤销错误版本                 |

## 3. 示例（基础）
```text
feat(flutter/im): 增加聊天列表页面懒加载

使用 ListView.builder 替换原有 Column 布局。
优化了长列表的内存占用，提升滑动流畅度。

Closes #12
```
