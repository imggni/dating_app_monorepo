# App 打包发布、热更新与合规规范

基于交友聊天 App 的开发方案，在进行项目打包、版本发布或处理国内应用商店合规性要求时，必须严格遵守以下流程与标准。

## 1. 合规性要求 (适配国内应用商店审核)
- **隐私合规**:
  - 用户首次启动必须弹出隐私政策弹窗，明示数据收集范围（手机号、头像等）。
  - 必须提供隐私政策详情页及撤回同意的入口。
  - 严禁违规获取或滥用敏感数据。
- **权限合规**:
  - 按需申请核心权限（如麦克风、相册、相机），申请前提供友好提示。
  - 权限被拒时，不能导致 App 崩溃或核心基础功能不可用。
- **账号合规**:
  - 必须实现基于手机号/短信验证码的真实注册与登录体系。
  - **强制要求**: 必须提供完整的账号注销功能，注销时需彻底清除数据库及缓存记录。
- **内容合规**:
  - 针对用户发布的内容（帖子、头像、聊天信息）实现基础敏感词/违规图片过滤。
  - 对接腾讯云内容审核 API。

## 2. 性能与体积优化
- **启动耗时**: 冷启动控制在 3 秒内，热启动 1 秒内。第三方 SDK（极光、腾讯 IM）的初始化必须进行延迟加载（放在首屏渲染后）。
- **包体积控制**:
  - Android 最终包体尽量 < 15MB；iOS 尽量 < 20MB。
  - 静态资源如音频、图片需进行强力压缩，使用 WebP 格式图片，剥离无用依赖。

## 3. 打包发布流程
- **Android**:
  - 生成并妥善保存自有签名证书（.jks），同一应用的不同版本签名必须一致。
  - 优先生成 `AAB` (Android App Bundle) 格式，辅以 `APK`。
  - **重要**: 打包前须采用腾讯云等工具进行应用加固，适配 64 位架构。
- **iOS**:
  - 配置对应的 p12 证书与描述文件（注意区分开发/测试/生产环境），匹配正确的 Bundle ID。
  - 内测通过 TestFlight 分发，生产环境提交至 App Store Connect。
- **内测分发**:
  - 采用 **蒲公英** 进行 Android/iOS 的团队内测与快速分发。

## 4. 热更新 (Hot Update)
- 集成 `flutter_downloader` + `flutter_webview_plugin` 实现热更新。
- **热更新原则**:
  - 仅用于更新 UI 细节、修复非核心逻辑的 Bug。
  - 严禁通过热更修改核心业务逻辑或增加需要新系统权限的功能，避免触发商店下架风险。
  - 更新包需支持加密并验证完整性。
