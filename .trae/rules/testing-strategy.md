# 测试策略（单元/集成/E2E）

**核心约束（必须遵守）**
1. **后端单元测试**: NestJS 中的核心业务逻辑（Service 层）必须编写 Jest 单元测试，覆盖正常逻辑与异常边界。
2. **后端集成测试**: 关键 API 接口（如登录、支付）需编写 e2e 测试，使用真实/测试数据库验证完整数据流。
3. **前端 Widget 测试**: Flutter 中可复用的核心 UI 组件需编写 Widget Test，确保 UI 渲染与交互正确。
4. **测试独立性**: 测试用例之间必须相互独立，不依赖执行顺序，测试结束后需清理脏数据。
5. **测试覆盖率**: 核心模块的代码覆盖率要求达到 80% 以上才允许合并 PR。
