# 错误处理规范

**核心约束（必须遵守）**
1. **后端异常抛出**: Express 中所有业务异常必须抛出内置的 `Error` 或自定义业务异常，严禁直接返回 HTTP 500 或程序崩溃。
2. **全局错误拦截**: 必须通过 Express 全局 `middleware` 中的 `errorHandler` 统一拦截异常，转换为标准 API 返回格式（code 非 0）。
3. **前端网络拦截**: Flutter 端必须在 Dio 拦截器中全局处理 HTTP 错误（401、403、500等）及业务错误码（code 非 0），并统一通过 Toast 提示用户。
4. **错误日志记录**: Express 中抛出的 Error 及前端捕获的未处理异常必须记录日志，包含堆栈信息与上下文参数。
5. **用户友好**: 错误提示必须面向用户（如“网络连接超时，请重试”），禁止将数据库报错等技术细节直接暴露给前端用户。
