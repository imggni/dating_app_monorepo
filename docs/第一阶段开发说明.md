# 第一次开发说明文档 (基于 NestJS + Flutter)

## 概述
本阶段开发基于《交友聊天App（Flutter+Express+CloudBase）完整开发方案.md》与项目本身环境和规范。
由于项目规范 (`.trae/rules/nestjs-specific.md`) 强制规定后端必须使用 **NestJS + TypeScript**，我们已经基于文档设计将其完整转换为 NestJS 实现（代替 Express）。

## 已开发内容

### 后端 (`dating_app_backend`)
1. **Prisma 模型重构**: 在 `prisma/schema.prisma` 中生成了对标设计文档中的全部数据模型：
   - `User` 和 `FriendRelation` (用户及好友)
   - `ImMessage` (即时通讯)
   - `SlowChat` (慢社交)
   - `GameRoom`, `GameRoomMember`, `GameRecord` (你画我猜)
   - `Circle`, `CirclePost`, `CircleComment` (兴趣圈子)
   - `SystemConfig` (系统配置)
2. **鉴权模块 (`AuthModule`)**:
   - `auth.controller.ts` 与 `auth.service.ts`：实现 `/api/auth/register` 与 `/api/auth/login`。
   - `dto/register.dto.ts` 和 `dto/login.dto.ts` 参数校验。
3. **用户模块 (`UserModule`)**:
   - `user.controller.ts` 与 `user.service.ts`：实现个人信息查询与更新 (`/api/user/profile`)。
4. **兴趣圈子 (`CircleModule`)**:
   - `circle.controller.ts` 与 `circle.service.ts`：实现了瀑布流列表、发布帖子、帖子详情、点赞和评论接口。
5. **认证守卫 (`DevAuthGuard`)**:
   - 补充 JWT 校验逻辑。

### 前端 (`dating_app_frontend`)
1. **网络请求层 (`lib/api/api_client.dart`)**:
   - 封装了 Dio 实例，统一配置超时和 JSON 返回格式。
   - 增加了 `InterceptorsWrapper` 拦截器，每次请求自动附加 `flutter_secure_storage` 中存储的 JWT Token，并全局拦截业务错误码。
2. **全局配置 (`lib/core/config/app_config.dart`)**:
   - 提取了环境中的 `imAppId` 与 `baseUrl`，方便环境切换。

## 待人为配置与解决项

为了跑通整个流程，需要人工进行以下操作：

1. **后端环境与依赖**:
   - 目前你电脑上的 npm/yarn 存在权限问题 (`npm error Your cache folder contains root-owned files`)，导致依赖未能成功安装。
   - **请在终端执行修复**: 
     ```bash
     sudo chown -R 501:20 "/Users/lianji/.npm-cache"
     sudo chown -R 501:20 "/Users/lianji/.yarn-cache"
     ```
   - 修复后，进入后端目录执行依赖安装和数据库客户端生成：
     ```bash
     cd dating_app_backend
     npm install
     npx prisma generate
     npx prisma db push  # 推送表结构到云数据库
     ```
2. **前端依赖**:
   - 前端已增加 `dio`, `flutter_secure_storage` 相关的代码，请确保在 `pubspec.yaml` 中包含这些包，并运行 `flutter pub get` 安装。
3. **环境变量**:
   - 你的 `.env` 已正确配置了 `DATABASE_URL` (MySQL) 和 `JWT_SECRET`，但后续在集成图片上传时需要配置 `CloudBase` / `腾讯云 COS` 密钥，集成推送时需要配置极光推送等秘钥。
4. **业务完善**:
   - 后端目前只搭建了 `Auth`, `User`, `Circle` 模块，其他模块（如 `Game`, `Im`, `SlowChat`）已建立好了数据结构，但需要进一步填充 Controller 与 Service 业务逻辑（特别是集成 Socket.IO 的部分）。
