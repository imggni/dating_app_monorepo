# 开发更新文档 (Auth & User Profile 功能对接 2026-04-22)

## 1. 概述
根据项目《交友聊天App（Flutter+nestjs+CloudBase）完整开发方案.md》的要求，本次更新主要完成了**用户认证（注册/登录）**以及**个人资料获取与更新**模块的前后端全链路功能对接。

## 2. 后端变更 (NestJS)
- **Prisma 数据库结构对齐**：确认了 `User` 表结构（包含 `phone`、`password`、`nickname`、`tags` 等字段）。
- **Auth 模块更新**：
  - 完善 `AuthController` 和 `AuthService`。
  - 实现了 `register` 接口：接收 `phone`、`password` 和 `nickname`，对密码进行 `bcrypt` 加盐哈希加密后存入数据库。
  - 实现了 `login` 接口：完成手机号与密码的校验，并签发 JWT Token，响应格式中统一返回 `access_token` 字段以便前端识别。
- **User 模块更新**：
  - 完善 `UserController` 和 `UserService`。
  - 实现了 `getProfile` 接口：通过 `jwt-auth.guard` 鉴权，根据解析出的 `userId` 查询并返回当前用户的详细资料（包含昵称、头像、简介、标签等）。
- **响应拦截器增强**：
  - `transform.interceptor.ts`：确保所有 Controller 的返回值自动包装成标准的 `{ code: 0, message: "success", data: {} }` 结构，与前端网络请求规范对齐。

## 3. 前端变更 (Flutter)
- **API 客户端生成**：
  - 基于 `Dio` 和 `retrofit` 新增了 `lib/api/user_api.dart` 接口声明，并运行 `build_runner` 生成了对应的 `user_api.g.dart`。
  - 包含了 `login`、`register`、`getProfile` 等具体的网络请求方法。
- **登录与注册页面对接**：
  - 更新 `lib/pages/login/login_controller.dart`：剥离旧的直接 Dio 调用，使用 `UserApi().login()`，并将获取的 `access_token` 保存到本地安全存储 `AppStorageService` 中。
  - 新增 `lib/pages/login/register_page.dart`：实现了注册表单UI与逻辑，调用 `UserApi().register()`，并在注册成功后自动返回登录页。
  - 在 `login_page.dart` 中加入了跳转注册页的入口按钮，并在 `app_routes.dart` 与 `app_pages.dart` 中注册了 `/register` 路由。
- **个人中心对接**：
  - 更新 `lib/pages/mine/mine_controller.dart`：在初始化时调用 `UserApi().getProfile()`，解析返回数据并更新 GetX 的 `profile` 响应式状态。
  - 增加了对后端返回的 `tags` 数据类型兼容处理（无论后端返回逗号分隔字符串还是数组，都能正确转换为 `List<String>` 供 UI 渲染）。
- **全局异常处理**：
  - 在所有请求的 `catch` 块中增加了针对 `DioException` 的拦截，并将后端抛出的具体错误信息（如“手机号已注册”、“密码错误”）直接通过 `Get.snackbar` 展示给用户。

## 4. 后续开发建议
- **后端**：在 `auth` 模块增加获取短信验证码逻辑及频控防刷。
- **前端**：增加修改个人资料的UI表单，并接入后端的 `updateProfile` 接口。
- **联调测试**：配置正确的 CloudBase 数据库连接串并在真实环境下进行联调。
