# 后端项目开发与更新文档

基于《交友聊天App（Flutter+nestjs+CloudBase）完整开发方案》的需求，针对 `/dating_app_backend` 进行的开发与更新如下。

## 一、本次更新内容概览

本次开发主要对齐了文档中定义的**五大核心模块**（用户、IM、游戏、慢社交、兴趣圈子）以及**通用模块**（文件上传、全局配置等），并完善了基于 NestJS + Prisma 的底层接口框架。

### 1. 数据模型适配
目前 `prisma/schema.prisma` 已经包含了文档中要求的所有核心数据表（如 `User`, `FriendRelation`, `ImMessage`, `SlowChat`, `GameRoom`, `Circle` 等），且关联关系已配置完整。

### 2. 用户模块 (User Module) 更新
在 `user.controller.ts` 和 `user.service.ts` 中补充了以下核心接口的结构与部分实现：
- `POST /v1/user/friend/request`: 发起好友请求
- `PUT /v1/user/friend/handle`: 处理好友请求
- `GET /v1/user/friend/list`: 获取好友列表
- `DELETE /v1/user/logout`: 账号注销
- `GET /v1/user/online/status`: 获取在线状态

### 3. IM 即时通讯模块 (IM Module) 开发
新增了 `im.controller.ts` 和 `im.service.ts`，提供了与前端腾讯 IM SDK 协同的后端接口规范：
- `POST /v1/im/message/save`: 消息同步存储
- `GET /v1/im/message/history`: 历史消息拉取
- `PUT /v1/im/message/read`: 消息已读状态同步
- `PUT /v1/im/message/recall`: 消息撤回同步
- `POST /v1/im/group/create`: 创建群聊（对接 IM SDK）

### 4. 游戏模块 (Game Module) 补充
新增和完善了 `game.controller.ts` 和 `game.service.ts`：
- `POST /v1/game/room/create`: 创建你画我猜房间
- `GET /v1/game/room/list`: 获取游戏房间列表
- `POST /v1/game/room/join` / `exit`: 加入和退出房间

### 5. 慢社交模块 (Slow Chat Module)
创建了慢社交接口规范，支持信件延迟发送：
- `POST /v1/slowchat/send`: 发送慢消息
- `GET /v1/slowchat/list`: 慢消息列表

### 6. 通用公共接口 (Common Business)
提供全局公用功能：
- `POST /v1/common/upload`: 文件上传至腾讯云 COS / CloudBase Storage
- `GET /v1/common/config`: 获取系统全局配置（如敏感词库、默认配置）

## 二、后续开发与维护计划

由于项目规模较大，部分功能（如 Redis 缓存强一致性、腾讯云 COS 直传签名、Socket.io 笔触同步细节、以及极光推送定时任务等）目前已预留了接口框架，后续需要根据前端实际联调情况进行深入对接：

1. **Redis 缓存集成**：后续需要在所有高频接口（如 `online/status`, `unread/count`）中引入 `@nestjs/redis` 提升性能。
2. **鉴权与守卫**：目前大部分接口挂载了 `DevAuthGuard`，生产环境需切换为基于 JWT 和 Passport 的完整 `JwtAuthGuard`。
3. **Socket.IO 联调**：在 `game.gateway.ts` 中处理前端发来的二进制压缩画板笔触数据，进行广播。
4. **统一异常拦截**：目前已包含 `http-exception.filter.ts`，未来需针对腾讯 IM SDK 及第三方调用的错误码进行映射转换。

> 提示：此文档作为交友聊天 App 后端迭代的跟踪日志，在后续版本迭代中需持续更新。
